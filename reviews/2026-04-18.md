## Risk flags

- **Zero test coverage.** No `test_*.py`, no `tests/` dir, no CI config. 2,690 lines of production code with nothing asserting behaviour. The agent tool-loop, SDK-to-JSON serialization (just patched), PDF parsing, search ranking, and dispatch layer are all untested.
- **Process-wide SSL monkey-patch.** `slack_app.py:28-31` mutates `os.environ["SSL_CERT_FILE"]`, `REQUESTS_CA_BUNDLE`, `WEBSOCKET_CLIENT_CA_BUNDLE`, and overrides `ssl._create_default_https_context` globally. Works for this standalone bot; will bite anyone importing the module into a larger process or test harness. Comment calls it a macOS/python.org fix — fine, but the scope is global and silent.
- **Broad `except Exception` in critical paths, swallowed silently:**
  - `slack_app.py:98` — fallback path in `_run_and_reply` catches every exception from the second `run_agent` attempt and falls through to a generic Dutch error message. No log line. If Anthropic is down, you won't know why from the logs.
  - `tools.py:53` — PDF page-extract failure is swallowed per page with no log; a scanned PDF silently yields the "no extractable text" fallback.
  - `tools.py:71` — `_read_text` top-level catch returns `"[Could not read file: …]"` as the file content, which then gets scored and ranked in `search_files`. A broken parser can look like a valid match.
  - `tools.py:240` — `web_search` swallows all DuckDuckGo errors into `{"error": …}`, which is treated as a normal tool result by the agent loop. Currently returning 119-char error payloads and the agent can't distinguish that from "no results."
- **Binary PDFs committed inline (~5.8 MB):** `01_Proposals/` holds three proposal PDFs totalling ~5.8 MB in the repo. Fine at n=3. At n=30 this repo is painful to clone. No LFS config, no guidance. Decide now, not later.
- **Dependency floors but no lockfile:** `requirements.txt` uses `>=` for all 8 deps. No `requirements.lock`, no `pip-tools`, no `uv.lock`. A reproducible build is not guaranteed. `duckduckgo-search` in particular is notoriously breaking between minor versions.
- **No schema migrations for `conversations.db`.** `memory.py:13-22` `CREATE TABLE IF NOT EXISTS` is the only schema logic. If the message format changes (it already has — see `_to_jsonable` at `memory.py:10-22` that was just added), existing rows become unreadable and will silently error out of `json.loads` in `load()` at line 31.
- **Auth surface:** Slack bot token is used directly as a bearer header in `slack_app.py:194` to download attachments. Fine for Slack-hosted files, but if `url_private` ever points elsewhere the token is leaked to a third party. No allowlist check on the host.
- **`search_files` reads every eligible file on every call** (`tools.py:172-188`). No caching. With PDFs now parsed via pypdf, each search iterates all ~30 source files through docx/pypdf/openpyxl. The Holland & Barrett run showed iterations 4-minute context-heavy loops; you'll hit this again on bigger source trees.
- **No TODO/FIXME left in code.** Good on this axis.
- **No disabled tests, no commented-out code blocks, no deletions.** Initial commit.
