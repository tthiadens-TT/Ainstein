Dit is onbereikbare code — de functie returned al daarvoor. Het fragment hoort waarschijnlijk thuis in de routing-functie die skill-namen bepaalt. Verwijderen of verplaatsen; nu doet het niets en het zaait verwarring bij toekomstige lezers.

**2. `_normalise_category` laat onbekende waarden door (`feedback.py`)**
De huidige implementatie:

```python
def _normalise_category(category: str | None) -> str:
    if category and category in ALL_CATEGORIES:
        return category
    return "missende-inhoud" if category is None else category  # ← onbekend doorgelaten
```

Als de classifier een waarde teruggeeft die *niet* `None` is maar ook *niet* in `ALL_CATEGORIES` staat (bijv. een model-variant die de labelset parafraaseert), wordt die ruwe string opgeslagen. De JSON schema enum in `record_correction` beschermt de inline-path; de Slack-classifier-path is afhankelijk van de regex-parsing in `_classify_feedback`. Als de LLM een categorie-label teruggeeft met een spatie of andere schrijfwijze, glipt het erdoor. Overweeg een strikte fallback: `return "missende-inhoud"` voor alle onbekende waarden.
