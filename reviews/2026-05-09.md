**`update_gdoc.py:37` — silent fail bij verlopen token zonder refresh_token (nieuw CF 24)**

`get_creds()` test `if creds.expired and creds.refresh_token:` — als het token verlopen is maar er is geen refresh_token, valt de code door en geeft een verlopen `Credentials`-object terug. De eerste Google API-aanroep geeft dan een HTTP 401 zonder duidelijke boodschap. Betere aanpak:

```python
if creds.expired:
    if creds.refresh_token:
        creds.refresh(Request())
        TOKEN.write_text(creds.to_json())
    else:
        print("ERROR: Token verlopen en geen refresh_token. Herrun setup_gdrive_auth.py.")
        sys.exit(1)
```

**`read_pdf.py:17-19` — malformed `--pages`-argument onafgevangen (nieuw CF 25)**

`--pages 3-` (trailing dash) levert `parts = ["3", ""]`, waarna `int("")` een `ValueError` gooit. De generieke `except Exception` in `__main__` vangt dit wel op, maar de foutmelding is cryptisch. Voeg toe: valideer dat `parts[1]` niet leeg is, of beschrijf het verwachte formaat in de error.

**`requirements.lock` — absoluut lokaal pad in pip-compile header (cosmetic)**

Het gegenereerde bestand bevat:
```
#    pip-compile --output-file=/Users/thomasthiadens/Ainstein/requirements.lock …
```
Dit absoluut pad klopt alleen op de lokale machine. Als een tweede persoon pip-compile uitvoert, overschrijft die het header. Overweeg `pip-compile` zonder `--output-file` te draaien vanuit de repo-root, of gooi de header weg in `.gitignore`-regels op dat bestand (niet mogelijk) — of accepteer dit als cosmetic.
