## For the developer [dev]

### NIEUW — `AINSTEIN_NOTES_DOC_ID` niet gedocumenteerd in `.env.example`

De finale `save_note` implementatie (commit `cbfb000`) vereist `AINSTEIN_NOTES_DOC_ID` in `.env`. Zonder deze variabele valt de tool terug op een lokaal `.md`-bestand — op de VM bestaat dat pad niet, waardoor de fallback ook faalt. Het ontbreken van deze variabele geeft geen foutmelding naar de gebruiker maar silently afval. `.env.example` bevat de variabele niet.

**Fix:** voeg toe aan `.env.example`:
```
# ID of the shared Google Doc for Ainstein notes (required for save_note).
# Create a doc as Thomas, share with the service account (Editor), copy the doc ID.
# AINSTEIN_NOTES_DOC_ID=1abc...xyz
```
Zet ook de variabele op de VM in `/home/thomas/Ainstein/.env`.

### NIEUW — `folder_hint` is dead code in `save_note`

Het tool-schema belooft: _"pass folder_hint to target a specific project folder (e.g. 'lead3', 'proposals')"_. De huidige implementatie in `_save_note_via_drive_api` negeert `folder_hint` volledig — alles gaat naar het enkelvoudige `AINSTEIN_NOTES_DOC_ID`-document. De `_FOLDER_HINT_MAP` in de eerdere dispatch-versie is er nog in commentaar-context maar werkt niet.

Keuze: (a) verwijder `folder_hint` uit het schema en de tool-beschrijving, of (b) implementeer het alsnog (bijv. als sectie-label in het document). Optie (a) is de eenvoudigste fix. Referentie: `tools.py:1587–1613` (schema) en `tools.py:1270` (implementatie).

### NIEUW — `slack_app.py` skill-detectie: "notes" is te breed

`slack_app.py:203` matcht op `"notes"` als enkel woord. Dit is een Engels woord dat in veel niet-archiverings-contexten kan voorkomen ("I have some notes on this proposal", "notes from last meeting" als context voor een analyse-vraag). De save_note check staat vóór `analyse_opportunity` en `build_proposal`, dus bij false positive krijgt de gebruiker geen antwoord maar een opslaginstructie.

**Overweeg:** verwijder `"notes"` als standalone trigger, of voeg context-check toe (bijv. alleen matchen als `"sla op"` of `"archiveer"` ook aanwezig is). Referentie: `slack_app.py:200–206`.

### NIEUW — `.env.example` comment onjuist: verwijzing naar `drive.file` scope

`.env.example` zegt: _"The service account must use drive.file scope"_. Na CF 23 (commit `a08a7757`, 18 mei) gebruikt de code de volledige `drive` scope. De comment is stale en misleidend voor iemand die de setup opnieuw doet.
