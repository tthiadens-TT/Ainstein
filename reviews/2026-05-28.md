### 5. `_cache_db()` has no lock â€” `tools.py:~60`

The lazy-init check `if _CACHE_DB is None` is not protected by `_READ_CACHE_LOCK`. Two concurrent Slack events can both see `None`, both call `sqlite3.connect()`, and the second connection overwrites the first. The DDL (`CREATE TABLE`, `PRAGMA`) runs twice on two separate connections; the overwritten connection leaks. **Fix:** wrap the `if _CACHE_DB is None` block with `_READ_CACHE_LOCK`.

### 6. `cmd_feedback_review` bypasses the rate limiter â€” `slack_app.py:~671`

All slash commands route through `_slash_handler`, which passes `user_id` from the Slack payload. `cmd_feedback_review` is the only slash handler that calls `_run_and_reply` directly with no `user_id`, so `_is_rate_limited("")` always returns `False`. A user can call `/feedback-review` unlimited times per hour while every other command is capped at `RATE_LIMIT_MAX`. **Fix:** add `user_id=body.get("user_id", "")` to the `_run_and_reply` call in `cmd_feedback_review`.
