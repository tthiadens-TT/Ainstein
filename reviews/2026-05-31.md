### New tool: `list_recent_files` — 2 minor issues

**PR #22 merged** (`tools.py:1533–1623`). The tool queries the Shared Drive for files created within a look-back window and optionally filters by creator name. Clean implementation with pagination and error handling. Two issues:

**[dev-1] Timestamp format missing `Z` suffix — `tools.py:1553`**

```python
cutoff = (datetime.now(timezone.utc) - timedelta(hours=hours)).strftime(
    "%Y-%m-%dT%H:%M:%S"  # should be "%Y-%m-%dT%H:%M:%SZ"
)
```

RFC 3339 requires an explicit timezone marker. Google Drive API is likely lenient in practice, but if it ever rejects the query or silently treats the timestamp as local time, files will be missed or returned incorrectly. Fix: add `"Z"` at the end of the format string.

**[dev-2] No results cap — `tools.py:1547`**

`_drive_list_recent_files` pages until `nextPageToken` is exhausted. A call with `hours=720` (one month, the example in the schema description) on a busy Drive could return hundreds of files and fill the LLM context window. Recommend adding a `limit` parameter (default 50) and stopping pagination when reached.
